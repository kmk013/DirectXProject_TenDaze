혹시나 비주얼스튜디오 버전 문제때문에 프로젝트 속성 초기화되면 이렇게 설정

[구성 : 모든 구성]
일반 -> 플랫폼 도구 집합 -> v120 (혹은 각자 버전에 맞게)
일반 -> 문자 집합 -> 유니코드 문자 집합 사용
VC++ 디렉터리 -> 포함 디렉터리 -> 편집 -> 
$(SolutionDir)dep\d3d\include 추가, $(SolutionDir)dep\fmod\include 추가
VC++ 디렉터리 -> 라이브러리 디렉터리 -> 편집 -> 
$(SolutionDir)dep\d3d\lib\x86 추가, $(SolutionDir)dep\fmod\lib 추가

그 외에도 추가하려는 라이브러리 있으면 솔루션 폴더의 dep 폴더에 넣어 include, lib, dll로 정리한 후 위와 같이 추가.

또한 함수에 _s 붙여서 쓰라는 오류 날 때엔 _s 붙인 함수로 검색해서 바꾸거나 C/C++ -> 일반 -> SDL검사 -> 예(/sdl) 로 변경

[구성 : Debug(디버그)]
C/C++ -> 코드 생성 -> 런타임 라이브러리 -> 다중 스레드 디버그(/MTd)

[구성 : Release(릴리즈)]
C/C++ -> 코드 생성 -> 런타임 라이브러리 -> 다중 스레드(/MT)


+ 모든 dll파일은 프로젝트 폴더 안에 있어야 함.
리소스파일은 기본 경로가 프로젝트 폴더이므로 프로젝트 폴더 안에 리소스 폴더를 만들어
(resources/image.png) 와 같이 경로설정가능.


배포할땐 release로 빌드하여 dll파일을 동봉해서 배포
(기본은 d3dx9_43.dll, fmodex.dll)
